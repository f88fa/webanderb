# دليل الرفع على الاستضافات المختلفة

## ✅ النظام التلقائي

المشروع الآن يدعم **اكتشاف تلقائي** للبيئة ويعمل على:

- ✅ **استضافة عادية** (Laravel Standard) - مجلد `public` منفصل
- ✅ **استضافة مشتركة** (Shared Hosting) - مثل هوستنجر - مجلد `public_html` مباشرة
- ✅ **تخزين تلقائي** - يقرأ المسارات تلقائياً في كلا الحالتين

---

## 🚀 طريقة الرفع

### للاستضافة العادية (Laravel Standard):

**لا حاجة لتغيير أي شيء!** المشروع سيعمل تلقائياً.

1. ارفع جميع الملفات
2. تأكد من أن `public` هو الجذر
3. المشروع سيعمل مباشرة

### للاستضافة المشتركة (مثل هوستنجر):

**لا حاجة لتغيير أي شيء!** النظام سيكتشف تلقائياً.

1. ارفع جميع الملفات إلى `public_html`
2. **لا حاجة** لنسخ محتويات `public` إلى الجذر
3. المشروع سيعمل مباشرة

---

## 📁 هيكل الملفات

### الاستضافة العادية:
```
project/
├── app/
├── public/          ← الجذر هنا
│   ├── index.php
│   └── .htaccess
├── storage/
└── ...
```

### الاستضافة المشتركة (هوستنجر):
```
public_html/         ← الجذر هنا مباشرة
├── app/
├── index.php        ← موجود في الجذر
├── .htaccess        ← موجود في الجذر
├── storage/
└── ...
```

**ملاحظة:** في الاستضافة المشتركة، جميع الملفات في `public_html` مباشرة.

---

## 🔧 الملفات المضافة

### 1. `index.php` (في الجذر)
- يكتشف تلقائياً نوع الاستضافة
- يعيد التوجيه للبيئة المناسبة

### 2. `.htaccess` (في الجذر)
- يكتشف تلقائياً وجود مجلد `public`
- يعيد التوجيه تلقائياً

### 3. `app/Helpers/PathHelper.php`
- Helper class للكشف التلقائي عن البيئة

### 4. `app/Helpers/helpers.php`
- Helper functions للاستخدام في الكود

---

## 📦 التخزين (Storage)

### يعمل تلقائياً في كلا الحالتين:

**في الاستضافة العادية:**
```
/storage/uploads/image.jpg
```

**في الاستضافة المشتركة:**
```
/storage/uploads/image.jpg
```

**نفس المسار!** النظام يكتشف تلقائياً.

## ✅ قائمة التحقق

### قبل الرفع:
- [ ] تحديث ملف `.env` ببيانات قاعدة البيانات
- [ ] تحديث `APP_URL` بالرابط الصحيح
- [ ] تشغيل `composer install --no-dev --optimize-autoloader`
- [ ] تشغيل `php artisan migrate --force`
- [ ] تشغيل `php artisan storage:link`

### بعد الرفع:
- [ ] اختبار الصفحة الرئيسية
- [ ] اختبار تسجيل الدخول
- [ ] اختبار رفع الصور
- [ ] اختبار عرض الصور المرفوعة

## 🔍 كيف يعمل النظام؟

### 1. الكشف التلقائي:
```php
// يكتشف تلقائياً:
- إذا كان في public_html → استضافة مشتركة
- إذا كان في public → استضافة عادية
```

### 2. التوجيه التلقائي:
```apache
# .htaccess يكتشف تلقائياً:
- إذا وجد public/ → يوجه إليه
- إذا لم يجد → يستخدم الجذر مباشرة
```

### 3. التخزين التلقائي:
```php
// المسارات تعمل تلقائياً:
/storage/uploads/image.jpg
// يعمل في كلا الحالتين!
```

## 🐛 استكشاف الأخطاء

### المشكلة: الصور لا تظهر
**الحل:**
```bash
php artisan storage:link
```

### المشكلة: خطأ 500
**الحل:**
1. تحقق من `.htaccess` موجود
2. تحقق من `index.php` موجود في الجذر
3. تحقق من الصلاحيات: `chmod -R 755 storage`

### المشكلة: المسارات خاطئة
**الحل:**
- النظام تلقائي، لا حاجة لتعديل
- إذا استمرت المشكلة، تحقق من `APP_URL` في `.env`
